# W6.6 — Silence driver hotfix

Mini-wave between W6.5 (W6 fix-sprint) and the W6.5 end-to-end demo.
Closes the "write-side has no source, so WRITE_REPLACE callback never
fires" hole that breaks bot audio playback on freshly-answered
channels with no active playback / bridge.

## Why this is needed

Per FF-001 (write-side single interleaved loop), a `WRITE_REPLACE`
callback fires only when FreeSWITCH has a `write_frame` to push
through the bug chain. The `write_frame` comes from one of:

- An active `playback` / `playseq` / `record_session` driver (FS apps
  like `mod_dptools::playback`, `mod_native_file`, `mod_sndfile`).
- A bridged leg's audio (the other party's mic).
- An IVR primitive (`switch_ivr_play_file`, `switch_ivr_say`, ...).
- Hold music (`mod_dptools::hold` with `moh_class` set).

On a "just answered + parked" channel with no other driver, FS sends
nothing to the write side — the bug callback is never invoked. The
bot's TTS audio queues in the module's `TtsPlayoutBuffer` /
`BugFanout` but never reaches the caller.

In the W6.5 demo dialplan we use `<action playback="silence_stream://-1"/>`
as the driver, but that's a workaround that requires operator-side
dialplan changes for every bot extension. Production operators will
forget. We need a module-owned driver.

## What W6.6 ships

When the module attaches an `SMBF_WRITE_REPLACE` bug to a channel
AND there is no other write-side driver active, **the module spawns
a one-shot thread** that calls
`switch_ivr_play_file(session, NULL, "silence_stream://-1", &args)`.
That call blocks forever (`-1` = infinite silence), producing the
write-side frames that the bug callback needs to fire.

On bot stop / channel hangup, the module sets
`switch_channel_set_flag(channel, CF_BREAK)` (FF-037). The play_file
loop checks the flag at each ptime tick and exits with
`SWITCH_STATUS_BREAK`. The driver thread joins and exits cleanly.

### Idempotency

If the operator's dialplan already drives the write side (any of the
sources listed above), the silence driver MUST NOT start a competing
thread. Detection rules:

1. **If a `playback` app is running** — check
   `switch_channel_test_flag(channel, CF_BROADCAST)` (FF: this flag
   is set by FS when a broadcast / playback is active). If set, skip.
2. **If the channel is bridged** — check
   `switch_channel_test_flag(channel, CF_BRIDGED)`. If set, skip.
   The bridge partner's audio drives the write side naturally.
3. **If a previous silence driver is already running for this
   channel** — track via `osw::media::SilenceDriverRegistry`. If
   present, skip.

The driver is attached opportunistically when (a) a bot WRITE_REPLACE
bug attaches AND (b) none of the above is true. It detaches when
the bot bug detaches (last bot bug detaching → driver also stops).

### `module_config.h` — 2 new fields

```cpp
struct ModuleConfig {
    // ... existing W6 fields ...

    // === W6.6 silence driver ===
    // Enable the silence driver.  Operators with always-bridged
    // bot calls can disable to save one thread per channel.
    bool silence_driver_enabled = true;

    std::uint32_t max_silence_drivers = 200;
```

### Audit subclasses

| Subclass | Tier | Trigger |
|---|---|---|
| `osw.media.silence_driver.started` | 2 | Per-channel driver thread spawned |
| `osw.media.silence_driver.stopped` | 2 | Per-channel driver thread exited (graceful or due to channel hangup) |
| `osw.media.silence_driver.cap_reached` | 1 | `max_silence_drivers` cap hit; new driver attach refused (operator-visible signal) |

Tier-2 events because driver lifecycle is normal flow; cap-reached is
Tier-1 because it indicates operator misconfiguration or pathological
load.

## Implementation steps

### 1. `osw::media::SilenceDriver`

`include/osw/media/silence_driver.h`:

```cpp
namespace osw::media {

class SilenceDriverRegistry;

/// One per-channel silence playback thread.  Spawned by
/// SilenceDriverRegistry::AttachOpportunistic when a WRITE_REPLACE
/// bug attaches and no other driver is detected.
class SilenceDriver {
 public:
    explicit SilenceDriver(std::string channel_uuid, SilenceDriverRegistry& reg);
    ~SilenceDriver();

    SilenceDriver(const SilenceDriver&) = delete;
    SilenceDriver& operator=(const SilenceDriver&) = delete;

    /// Set CF_BREAK flag on the channel + join the thread.  Idempotent.
    void Stop() noexcept;

    bool IsRunning() const noexcept;

 private:
    void Run() noexcept;          // thread body

    std::string channel_uuid_;
    SilenceDriverRegistry& reg_;
    std::thread thread_;
    std::atomic<bool> running_{true};
};

class SilenceDriverRegistry {
 public:
    explicit SilenceDriverRegistry(const ModuleConfig& cfg);
    ~SilenceDriverRegistry();

    /// Called from MediaBugManager::Attach when a WRITE_REPLACE bug
    /// just attached.  Spawns a driver if (a) config allows,
    /// (b) channel has no existing driver,
    /// (c) channel is not bridged,
    /// (d) channel is not broadcasting,
    /// (e) cap not reached.
    /// Idempotent — repeat calls are no-ops.
    void AttachOpportunistic(switch_core_session_t* session) noexcept;

    /// Called from MediaBugManager::DetachInternal when the last
    /// WRITE_REPLACE bug on a channel detached.
    void DetachIfOrphan(std::string_view channel_uuid) noexcept;

    /// Called from CS_DESTROY state hook (channel hangup).
    void RemoveForChannel(std::string_view channel_uuid) noexcept;

    /// Called from Module::Shutdown.
    void DrainAll();

    /// Stats.
    std::size_t ActiveCount() const noexcept;

 private:
    const ModuleConfig& cfg_;
    mutable std::mutex mu_;
    std::unordered_map<std::string, std::unique_ptr<SilenceDriver>> drivers_;
};

}  // namespace osw::media
```

`SilenceDriver::Run()`:

```cpp
void SilenceDriver::Run() noexcept {
    osw::raii::SessionLock sess(channel_uuid_);
    if (!sess) {
        // Channel disappeared between Attach decision and thread start.
        running_ = false;
        return;
    }
    switch_input_args_t args = {};
    args.input_callback = nullptr;
    args.buf = nullptr;
    args.buflen = 0;
    args.read_frame_callback = nullptr;
    args.user_data = nullptr;
    args.dmachine = nullptr;
    args.loops = 0;
    // Blocks until CF_BREAK is set (FF-037).  silence_stream://-1
    // produces infinite silence; the IVR loop checks CF_BREAK at
    // each ptime tick.
    switch_status_t st = switch_ivr_play_file(
        sess.session(),
        /*fh=*/nullptr,
        /*file=*/"silence_stream://-1",
        &args);
    (void)st;  // SWITCH_STATUS_BREAK is the expected exit code
    running_ = false;
}

void SilenceDriver::Stop() noexcept {
    osw::raii::SessionLock sess(channel_uuid_);
    if (sess) {
        switch_channel_set_flag(sess.channel(), CF_BREAK);
    }
    if (thread_.joinable()) thread_.join();
}
```

`SilenceDriverRegistry::AttachOpportunistic`:

```cpp
void SilenceDriverRegistry::AttachOpportunistic(
        switch_core_session_t* session) noexcept {
    if (!cfg_.silence_driver_enabled) return;
    if (!session) return;
    const char* uuid_c = switch_core_session_get_uuid(session);
    if (!uuid_c) return;
    const std::string uuid(uuid_c);

    switch_channel_t* chan = switch_core_session_get_channel(session);
    if (!chan) return;
    if (switch_channel_test_flag(chan, CF_BROADCAST)) return;  // playback active
    if (switch_channel_test_flag(chan, CF_BRIDGED)) return;    // bridged

    std::lock_guard<std::mutex> lk(mu_);
    if (drivers_.find(uuid) != drivers_.end()) return;          // already running
    if (drivers_.size() >= cfg_.max_silence_drivers) {
        osw::audit::Emit("osw.media.silence_driver.cap_reached",
                         osw::events::Tier::k1Durable,
                         {{"Unique-ID", uuid},
                          {"cap", std::to_string(cfg_.max_silence_drivers)}});
        return;
    }
    drivers_[uuid] = std::make_unique<SilenceDriver>(uuid, *this);
    osw::audit::Emit("osw.media.silence_driver.started",
                     osw::events::Tier::k2State,
                     {{"Unique-ID", uuid}});
}
```

### 2. `MediaBugManager` integration

In `MediaBugManager::Attach`, after a successful bug add:

```cpp
// W6.6: opportunistic silence driver attach when WRITE_REPLACE.
if (cfg.flags & SMBF_WRITE_REPLACE) {
    silence_driver_registry_.AttachOpportunistic(session);
}
```

In `MediaBugManager::DetachInternal`, after removing the bug record:

```cpp
// W6.6: if no more WRITE_REPLACE bugs on this channel, stop the driver.
if ((removed_flags & SMBF_WRITE_REPLACE) && !HasWriteReplaceBug(channel_uuid)) {
    silence_driver_registry_.DetachIfOrphan(channel_uuid);
}
```

The CS_DESTROY state hook (closed by W6.5 P1-003 fix) calls
`silence_driver_registry_.RemoveForChannel(uuid)` alongside the
existing `MediaBugManager::DetachAll(uuid)`.

## Out of scope

- Driving the **read** side. FS handles caller mic delivery; the
  module never needs a read-side driver.
- Replacing the silence with hold music when the bot is silent. V2
  may want this for UX but V1 keeps it simple.
- Per-tenant configuration of `silence_driver_enabled` — module-
  wide only in V1.

## Dependencies

- **W6.5 fix-sprint** — must be merged. The silence driver relies on
  `RegisterStateHandlers()` working (W6.5 P1-003 fix) for the
  CS_DESTROY teardown path. If the state hook is still a stub, the
  driver thread leaks on channel hangup.
- Module `mod_tone_stream` must be loaded by FreeSWITCH (provides
  `silence_stream://` schema). Already required in the demo Docker
  image's modules.conf.xml; production base image must include it.
