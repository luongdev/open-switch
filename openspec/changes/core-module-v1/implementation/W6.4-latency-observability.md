# W6.4 — Latency observability gate

W6.4 exists to make W6.5 measurable.  A Linphone call that can hear
the bot is not enough to close the demo gate if the pipeline cannot say
where the first-audio delay came from.

## Goal

For every TTS / voicebot turn, produce one correlated trace across:

```text
open-switch StartBot
  -> tts-edge OpenSwitch bridge
  -> tts-engine SynthesizeTurnStream
  -> tts-edge 20 ms frame egress
  -> open-switch StreamClient receive
  -> TtsPlayoutBuffer first pop
  -> RTP first non-silence packet (when pcap is enabled)
```

The trace must identify whether a miss was caused by:

- llama first-token / first-N-token latency
- codec first decode latency
- engine-to-edge streaming overhead
- edge resample / framing overhead
- edge-to-open-switch MediaBridge overhead
- open-switch jitter/preroll buffering
- FreeSWITCH / RTP media path

## Trace identity

Every span / structured log / metric point must carry enough labels to
join the turn:

- `trace_id` — UUIDv7 minted by the caller of `StartBot` or by
  `tts-edge` when the legacy path has none
- `turn_id`
- `bot_id` when available
- `stream_id`
- `channel_uuid`
- `tenant_id`
- `engine_profile` fields: codec backend, device, real/stub,
  `codec_streaming_frames_per_chunk`, eager-init/warmup state,
  cache enabled

Cross-process wall-clock timestamps are for log correlation only.
Latency SLOs are computed by the owning process with monotonic clocks.

## Required milestones

### tts-engine

- `engine.turn.received`
- `engine.commit.received`
- `engine.voice_lookup.done`
- `engine.normalize.done`
- `engine.prompt.done`
- `engine.tokenize.done`
- `engine.llama.first_token`
- `engine.llama.first_codec_chunk_ready`
- `engine.codec.first_decode.start`
- `engine.codec.first_decode.done`
- `engine.first_pcm.yield`
- `engine.last_pcm.yield`

### tts-edge

- `edge.stream_start.received`
- `edge.engine_stream.open.start`
- `edge.engine_stream.commit.sent`
- `edge.engine.first_pcm.received`
- `edge.resample.first_frame.done`
- `edge.first_audio_frame.sent_to_osw`

### open-switch

- `osw.start_bot.received`
- `osw.media_stream.open.start`
- `osw.media_stream.ready`
- `osw.stream_client.first_audio.received`
- `osw.tts_buffer.first_push`
- `osw.tts_buffer.preroll_reached`
- `osw.tts_buffer.first_audio_pop`
- `osw.write_replace.first_set_frame`

### External / optional

- `rtp.first_packet`
- `rtp.first_non_silence_packet`

## Metrics

Histograms use buckets:

```text
0.01, 0.02, 0.05, 0.08, 0.1, 0.15, 0.25, 0.5, 1, 2, 5
```

Required metric names:

- `tts_engine_ttfb_seconds`
- `tts_engine_llama_first_token_seconds`
- `tts_engine_llama_first_codec_chunk_seconds`
- `tts_engine_codec_first_decode_seconds`
- `tts_engine_pcm_gap_seconds`
- `tts_edge_engine_first_pcm_seconds`
- `tts_edge_first_frame_send_seconds`
- `osw_stream_first_audio_seconds`
- `osw_tts_buffer_preroll_seconds`
- `osw_tts_first_pop_seconds`
- `osw_audio_gap_seconds`
- `osw_tts_underruns_total`

Labels must stay bounded.  Do not put raw text, prompt text, or UUIDs
on Prometheus histograms.  UUIDs belong in structured logs only.

## Implementation tasks

1. **OpenSpec docs**
   - Add this W6.4 plan.
   - Link W6.5 acceptance to W6.4 metrics.
   - Add matching `open-tts` observability notes under the
     `realtime-local-model-tts-service` change.

2. **tts-engine instrumentation**
   - Add structured trace events for the required engine milestones.
   - Add histograms for first token, first codec chunk, first decode,
     first PCM, and PCM gaps.
   - Ensure `codec` phase metrics do not include llama token wait time
     when streaming is pull-based.
   - Avoid accumulating PCM chunks when sentence cache is disabled.
   - Add regression test: first PCM emits before llama stream completes.

3. **tts-edge instrumentation**
   - Log/measure first engine PCM received.
   - Log/measure first 20 ms frame sent to `mod_open_switch`.
   - Include bridge profile: output rate, resampler, frame duration,
     engine endpoint.

4. **open-switch instrumentation**
   - Log/measure first `FromService.audio` received by `StreamClient`.
   - Log/measure first push and first pop in `TtsPlayoutBuffer`.
   - Log/measure first WRITE_REPLACE frame actually set.
   - Preserve existing audio quality behavior; W6.4 must not add
     playout latency.

5. **Benchmark report**
   - Report profile fields: backend, device, real/stub, chunk size,
     eager-init/warmup, cache state, sample rate, FS channel rate.
   - Print a per-turn waterfall with the milestone durations above.
   - Fail the W6.5 gate when confirmed synthesis TTFB is above 250 ms
     or p95 audio gap is above 80 ms.

## Acceptance

- Unit/regression tests prove streaming first PCM can emit before
  llama stream completion.
- Local benchmark report includes a per-turn waterfall and profile.
- W6.5 Linphone test can cite:
  - engine first PCM
  - edge first frame
  - open-switch first audio receive
  - buffer first pop
  - RTP first non-silence when pcap is enabled
- Gemini CLI review is run on the implementation diff before merge.
